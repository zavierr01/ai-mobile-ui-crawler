Create `src/mobile_crawler/domain/context_guard.py` with:

1. `DeviceContext` dataclass with fields: `package: str`, `activity: str`, `is_target_app: bool`, `captured_at: datetime`
2. `DeviceContextCapture` class that takes `target_package: str` and `adb_executor: ADBActionExecutor` in `__init__`. Method `async capture(self) -> DeviceContext` that calls `adb_executor.get_current_package()`, `adb_executor.get_current_activity()`, compares package against `target_package` to set `is_target_app`, and returns a `DeviceContext`. Method `get_context_dict(self, ctx: DeviceContext) -> dict` returns `{"current_package": ctx.package, "current_activity": ctx.activity}` for persistence.

In `adb_action_executor.py`, make two changes:

A) **Fix `get_current_package()` pipe bug**: The current code passes `|` as a literal argument to subprocess, but the pipe should be inside the shell command. Change from `['shell', 'dumpsys', 'window', '|', 'grep', '-E', 'mCurrentFocus|mFocusedApp']` to `['shell', 'dumpsys window | grep -E mCurrentFocus']` — the entire shell command string is a single argument to `adb shell`. This matches the pattern used in DroidRun's `_get_current_app()`. Also fix the parser: instead of splitting on spaces and looking for `/`, use `re.search(r'([a-zA-Z0-9_.]+)/([a-zA-Z0-9_.]+)', output)` to extract the package (group 1). Add `import re` at the top.

B) **Add `get_current_activity(self) -> Optional[str]` method**: Similar to the fixed `get_current_package()`, run `adb shell 'dumpsys window | grep -E mCurrentFocus'` and extract both package and activity. Return the activity component (the part after `/`). Example: for `mCurrentFocus=Window{abc com.example.app/com.example.app.MainActivity}` return `com.example.app.MainActivity`. If activity extraction fails, return `None`.
