2. `UIDumpValidator` class with methods:
   - `validate(self, ui_data: Any) -> UIDumpValidationResult` — Takes raw UI data (could be list, dict, XML string, or None). Returns validation result. Validation logic per D-03: check (a) parseable — for XML strings, try `xml.etree.ElementTree.fromstring()`; for list/dict, check it's a non-None iterable; (b) non-empty — for lists: `len(data) >= 1`; for dict: check a11y_tree or elements key has items; for XML: root has at least 1 child element. On parse failure, return `is_valid=False, is_parseable=False, error=...`. On empty dump, return `is_valid=False, is_parseable=True, element_count=0, error="empty_ui_dump"`.

3. `validate_ui_dump_with_retry(self, ui_data_getter: Callable, max_retries: int = 1) -> UIDumpValidationResult` — Per D-04: calls `ui_data_getter()` to get fresh data, validates with `validate()`. If invalid and retries remain, calls getter again (one retry for transient failures). If still invalid after retry, returns the invalid result. Never raises an exception — always returns a result object.

Add a `StepSkipReason` enum in context_guard.py: `TARGET_APP_MISMATCH = "target_app_mismatch"`, `INVALID_UI_DUMP = "invalid_ui_dump"`. This enum will be used by both Plans 02 and 03.

Add `import xml.etree.ElementTree` and `from typing import Callable, Any, Optional, List` at the top of the file.
